# 半结构化数据模型Semi-structured Data Model

所有者: H34V3N

记录型模型一定要记录特定的类型的数据，缺一不可

半结构化模型就是一种“模棱两可的记录型模型”，他仍然采用类型-属性的模式记录数据，但是并**不要求记录在内的条目拥有完全相同的类型以及属性**，这一条有对长度的描述，下一条就可以没有

也就是说，只要你按每个条目保存数据（就像记录模型的中的“记录”），条目内部的数据都能**准确描述一个类型**，就行了

示例：

（姓名：李华，性别：男，学号：123456789，专业：计算机科学与技术）、

（姓名：李明，性别：男，专业：土木）

这个数据库就是半结构化数据模型，可以看到，里面记录的每个数据都知道自己在描述什么，但可以看到，第一条有学号，但是第二条没有

（姓名：李华，性别：男，学号：123456789，专业：计算机科学与技术）

（姓名：李明，性别：男，专业：土木，1231231，哈基米哈基米哈基米）
