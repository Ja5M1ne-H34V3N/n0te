现有一个学生在学校里启动一台笔记本电脑，请求和接收google官网
# 得到ip的分配

首先，笔记本知道了谷歌的域名，就还需要根据这个域名得到对应的ip地址
DNS的地址，下一跳的路由器的ip地址以及自己的ip地址采用DHCP（动态主机配置协议，应用层协议，理解为找服务器给自己发一个ip）得到，那么首先就要找到DHCP的服务器
所以笔记本则进行DHCP请求，将请求封装在UDP（传输层），IP（网络层），803.2的以太网帧（链路层）中，在局域网之内广播，被DHCP服务器接收到,服务器返回信息向电脑回应，电脑再发送申请ip的请求，DHCP收到信息后，生成DHCP ACK，，包括这个笔记本的端IP地址，第一跳路由器的IP地址和DNS服务器的地址，发送给目标笔记本主机

# 得到下一跳路由器的MAC地址
首先笔记本带着下一跳路由器的IP进行局域网内的ARP query广播（who has 路由器ip，tell 笔记本IP）
目标路由器在收到这个消息后，将MAC“地址信息发给笔记本
笔记本就知道了下一跳路由器的MAC地址，将把DNS查询发给下一跳路由器
# 得到谷歌的ip地址
接下来，笔记本电脑尝试访问DNS服务器得到谷歌的IP地址
笔记本创建一个DNS查询，封装在UDP IP 以太网帧中，发给下一跳路由器
以太网帧通过局域网转发，到达下一跳路由器，再经过路由器转发多次，到达DNS服务器
DNS解包这个DNS请求，把谷歌的IP地址发给这个笔记本

# 向谷歌发送消息，打开到达WEB服务器的TCP Socket
得知了谷歌的IP之后，笔记本编写用来建立连接的三次握手报文，接下来将第一次握手SYN段按同样的域间路由方式送到谷歌的web服务器
谷歌的web服务器在收到后，回复第一次握手的TCP SYNACK

之后完成第三次握手，笔记本和web服务器的TCP链接就建立了
